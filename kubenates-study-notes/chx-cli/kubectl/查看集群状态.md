## kubectl cluster-info

查看集群的信息。

```
root@cicd-server ~]# kubectl cluster-info
Kubernetes master is running at https://192.168.160.39:6443
KubeDNS is running at https://192.168.160.39:6443/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-info dump'.
```

## kubectl get cs (kubectl get componentstatuses)

获取各个组件的状态。

```
[root@cicd-server ~]# kubectl get cs
NAME                 STATUS    MESSAGE             ERROR
scheduler            Healthy   ok
controller-manager   Healthy   ok
etcd-0               Healthy   {"health":"true"}
```
